信息搜集

发表于 2022-03-03 分类于 渗透测试 Valine：

信息搜集的意义以及常用的一些信息搜集工具。

信息搜集的意义

为什么要信息搜集

找到新的域名/找到新的资产（子域名）/找到新的突破点（端口）

渗透测试：寻找安全问题

渗透测试=>信息搜集=>搜集薄弱资产

信息搜集的方向

  1. whois

    注册域名时留下的信息（直接百度whois就可以查,在里面的whois反查可以找到他注册的其他域名) whois（这个网站是最权威的，是国外的网站，可以看到完整的信息，邮箱，姓名等），当搜到一个公司的时候，还可以用天眼查查一查这个公司的详细信息

    社工型字典 可以通过填写一些信息猜测别人的密码，配合burp渗透更好用

  2. 子域名

    （有概率是同一台服务器）子域名一般属于同一个公司的，可能有的子域名防护得没那么严

    tips:子域名和旁站没有关系

    how?

    探测子域名：爆破DNS(用layer软件)

  3. 旁站

    同IP的站点 www.a.com =>1.1.1.1 www.b.com =>1.1.1.1(旁站) 但同ip不代表同服务器（分两种情况：1.同ip同服务器;2.同ip不同服务器但同一个内网)站长工具可以查，现在有种技术叫CDN加速，如果网站通过CDN加速，则不是真实的ip，可以通过指令ping 来看，如果不断变化则使用了CDN

    如何绕过CDN?

    • 内部邮箱源。一般邮件都在系统内部，没通过CDN，通过注册或者RSS订阅收到的邮箱查找
    • 子域名。一般只有主站挂CDN，而子站没有
    • 国外访问（从国外ping），一般都只在国内挂CDN，国外不一定
    • 查询域名的解析记录查询历史记录

  4. 目录扫描

    Why？有些网站可能某个目录下是一个新的网站，有的时候目录扫描直接下载了压缩包源码、编辑器目录、一些废弃的页面（会报错）

    How？（用御剑）要拿下一个站点要找后台（功能点），可以通过目录扫描，并发线程数调低一点，不要把别人站扫崩了（严格的就改成1比较好）常见的有/admin/login.asp

    /robots.txt君子协议，对方希望你不去访问

  5. 端口探测(Nmap)

    介绍

    为什么要端口探测？

    探测服务（他架设了网站，开启了远程桌面，SMB共享,开启了外联数据库）

    有些危险端口开放了我们就可以尝试入侵，例如445|3306|22|1433|6379 可以尝试爆破或者是使用某些端口存在漏洞的服务。（产生问题的不是端口，是端口所对应的服务）而且有可能一台服务器上面不同端口代表着不同的Web网站

    有域名的防护一般比没域名的强，80或者443默认端口的一般防护比其他端口强，所以说，我们找没有域名的非80 443 端口的站攻击可能更容易

    高危端口：

    | 21 | FTP |
    | :—: | :——————-: |
    | 22 | ssh |
    | 3306 | mysql |
    | 8080 | 其他WEB站 |
    | 6379 | redis未授权访问 |
    | 1433 | MSSQL(爆破) |
    | 3389 | RDP（0708） |
    | 445 | smb（永恒之蓝） |

    这些是默认端口，是可以改的

    how

    用nmap

    先用站长工具里的ping检测找到域名的ip，它会从各个地方访问，如果都是同一个ip则是真实ip

    然后在cmd用nmap ip地址 扫描，会得到一些信息

    扫出来后可以去尝试是否是web端口/远程桌面端口等

    nmap -p 80,443 只扫描80,443的端口

    nmap -o ip地址 可以探测目标的操作环境（什么系统）

    nmap -Pn 跳过主机发现过程直接端口扫描 [目标禁ping时]

    nmap -A 全部扫描，包括一切探测

    nmap -v 显示扫描过程

    对ip进行全扫描可以用nmap -p 1-65535 ip地址

    也可以用在线工具进行端口扫描端口扫描

    如果有高危端口，可以直接用永恒之蓝拿下

    tips:看看君子协议robots.txt

  6. C段扫描

    175.21.55.6和175.21.55.7就是同一c段，即只有最后一个点后的数字不同，赌他们是同一内网，但是很容易误伤，不建议使用

  7. 谷歌语法 谷歌镜像站

    | 命令 | 作用 |
    | ——————— | ——————————————- |
    | site:xxx | 指定域名 |
    | inurl:xxx | 指定URL栏 |
    | filetype:xxx | 数据类型(doc/xls/conf/inc…) |
    | “xxx” | 双引号里内容不能拆分 |
    | intitle:xxx | 网页标题中的关键字 |
    | link:xxx | xxx的友情链接 |
    | Info | 查找指定站点的一些基本信息 |
    | intext:xxx | 指定内容 |

    比如site:xxx 后台/管理/admin/login ，可能可以找到后台(域名写顶级域名访问的范围更广)

inurl:id/php/aspx/do/action/admin （靠平时积累）

伪静态（假的静态）：
